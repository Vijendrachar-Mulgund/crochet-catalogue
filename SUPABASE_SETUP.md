# Turn Charming Yarns into a shared online catalogue

Follow these steps once. After this, you and your mother can both log in from
any computer and build the same catalogue together. It's free.

You'll do three things: (1) create a free database, (2) create your login,
(3) paste two values into the app. Then see `DEPLOY.md` to put it online.

---

## 1. Create a free Supabase project

1. Go to **https://supabase.com** and click **Start your project** → sign up
   (you can sign in with GitHub or an email — no credit card needed).
2. Click **New project**.
   - **Name:** Charming Yarns
   - **Database Password:** click *Generate a password* and **save it somewhere**
     (you won't need it day-to-day, but keep it safe).
   - **Region:** pick the one closest to you (e.g. *South Asia (Mumbai)*).
3. Click **Create new project** and wait ~2 minutes for it to finish setting up.

---

## 2. Create the catalogue's tables

1. In the left sidebar click **SQL Editor** → **New query**.
2. Copy **everything** in the box below, paste it in, and click **Run**.
   You should see "Success. No rows returned".

```sql
-- Settings (single row holds business name, tagline, logo)
create table if not exists settings (
  id            int primary key default 1,
  business_name text default '',
  tagline       text default '',
  logo_data_url text
);

create table if not exists categories (
  id         bigint generated by default as identity primary key,
  name       text not null,
  sort_order int  default 0
);

create table if not exists products (
  id              bigint generated by default as identity primary key,
  name            text not null,
  category_id     bigint references categories(id) on delete set null,
  price           numeric,
  description     text default '',
  materials       text default '',
  dimensions      text default '',
  made_to_order   boolean default false,
  production_time text default '',
  colour_options  text default '',
  photo_data_url  text,
  created_at      timestamptz default now()
);

-- Security: only people who are logged in can read or change anything.
alter table settings   enable row level security;
alter table categories enable row level security;
alter table products   enable row level security;

create policy "signed-in full access" on settings
  for all to authenticated using (true) with check (true);
create policy "signed-in full access" on categories
  for all to authenticated using (true) with check (true);
create policy "signed-in full access" on products
  for all to authenticated using (true) with check (true);
```

---

## 3. Create your shared login

1. In the left sidebar click **Authentication** → **Users** → **Add user** →
   **Create new user**.
2. Enter an **email** and a **password** you'll both use (e.g. the business
   email). **Tick "Auto Confirm User"** so you can log in straightaway.
3. Click **Create user**.

(This is the email + password you'll type on the catalogue's sign-in screen.)

---

## 4. Copy your two keys into the app

1. In the left sidebar click **Project Settings** (gear icon) → **API**.
2. You need two values from this page:
   - **Project URL** (looks like `https://abcdefg.supabase.co`)
   - **anon public** key (a long string under *Project API keys*)
3. Open the file **`js/config.js`** in this folder and paste them in:

```js
window.CATALOGUE_CONFIG = {
  supabaseUrl: "https://abcdefg.supabase.co",     // your Project URL
  supabaseAnonKey: "eyJhbGciOi...your-long-key",  // your anon public key
};
```

Save the file. That's it — the app is now in shared online mode and will ask
you to sign in.

> These two values are safe to share/publish. They only allow what the
> security rules above permit, and changing the catalogue still requires
> logging in with the email + password from step 3.

---

## 5. Put it online

See **`DEPLOY.md`** to publish the site and get a link like
`charming-yarns.netlify.app` that you can send to your mother.

## Everyday use
- Open the link, sign in once (your browser remembers you).
- Add products and photos — they save to the shared database instantly.
- The other person sees new items the next time they open or refresh the page.
